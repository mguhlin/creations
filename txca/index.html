Main Navigation Bar (Left → Right)

Mastering the Knowledge Bank

▶
The "Water in a Glass" Analogy
An AI's context window is like a glass of water. The documents you upload take up space — keep them modular and relevant. One topic per file whenever possible.

Supported File Formats

Knowledge Bank accepts: .pdf, .docx, .txt, .csv, .png, .jpg, .xlsx — up to 100 MB per file. Text files (.txt) are the most token-efficient format for the AI to process.

Adding Knowledge

  • Drag and drop files directly into the chat window
  • Paperclip icon → "Attach" to upload or "Add Knowledge to Bank" to store permanently
  • When knowledge is attached, a number appears next to the paperclip showing how many files are active
  • All bots and all members in the chat can reference attached knowledge — but members cannot download your files

The Knowledge Bank

  • All uploaded files are automatically stored in your personal Knowledge Bank (brain icon)
  • Files are encrypted — no third parties can access without your explicit authorization
  • Organize, rename, filter, and delete files at any time
Pro Tip — Starring Items
Open the Knowledge Bank (brain icon). Click the Star next to a file to automatically attach it to every new chat. Perfect for custom instructions, grading rubrics, or district policies you use daily.
Pro Tip — MY PREFERENCES File
Create a text file called "MY PREFERENCES" that contains your name, role, school/organization, writing style preferences, and any standing instructions. Star it. Every new chat starts with BoodleBox already knowing who you are.
Pro Tip — Saving Chats as Knowledge
Use the three-dot menu on a successful chat and select "Add chat to knowledge bank" — future bots can learn from past successful outputs. Chat exports appear with a speech-bubble icon in your Knowledge Bank.

Modes & Collaborative Group Chats

▶

BoodleBox Modes

🤖 Bot Mode

Standard 1-on-1 interaction with an AI. The default mode for all new chats.

💬 Message Mode

Communicate with human team members. Type / before a message in the chat bar to switch, or use the three-dot menu → "Switch to Message Mode." Switch back the same way.

📄 Knowledge to Context Mode

Forces the AI to strictly use only attached documents. Enable via the three-dot menu in the chat bar. Use when the entire content of a document is required — complex analyses, pattern identification across documents. Zero hallucination risk.

🎓 Coach Mode

AI teaches you how to use the platform and build better prompts. Great for onboarding new staff or students.

The Magic of Group Chats

Multiple students (or colleagues) can be in a BoodleBox group chat with one or more AI bots simultaneously. Share via link through email, Slack, or your LMS.

Teacher Superpower
Teachers can jump into the chat to offer suggestions. The chatbot cannot see the teacher's participation — only the students can. This provides a transparent, auditable record of student thinking and prompt refinement. Teachers can monitor folders to view total chats, prompts, messages, and bots per student.
Classroom Feature — Folders
Instructors can share a folder link with an entire class. BoodleBox records individual and group activity inside — useful for assessing AI usage, identifying participation gaps, and applying rubrics. Add any folder or chat to your Knowledge Bank and ask a bot to identify learning gaps.

Bot Stacking & App Smashing

▶

Bot Stacking = "app smashing" within a single interface. Instead of copying/pasting between websites, you give the AI a to-do list and switch models mid-conversation. The entire chat history carries forward — every bot you switch to can see what happened before.

Live Workflow Demonstration

  1. Heavy Lifting: Use Gemini 3 Pro or ChatGPT 5 to summarize a large uploaded document. These handle massive context best.
  2. Refining: Switch to Claude 4.5 Sonnet in the same chat to turn that summary into a lesson plan, rubric, or quiz. Claude excels at structured writing.
  3. Fact-Check: Switch to Perplexity — type @ in the chat bar, select Perplexity. Ask it to verify claims from Step 2 with citations. Click the globe icon to see sources.
  4. Visuals: Switch to Nano Banana Pro or Flux 2 Pro to create a graphic for the lesson — no watermarks, no design skills required.

Pro Tip — Selecting Bots
Open the Bot Selector dropdown in your chat bar. Models, BoodleBots, and custom bots are separated by category. Use the search box with a keyword (e.g., "marketing," "rubric") to filter instantly. Three-dot menu on any bot → View Bot Profile to learn its expertise before switching.
Bot vs. Chat + Knowledge Bank
A chat with Knowledge Bank attached is similar to a Claude Project or Perplexity Space — context for a conversation. A custom bot is more like a Custom GPT — it has its own instructions, knowledge, and persona that persist across all conversations. Both are powerful; know which you need.

Deep Dive

Building Your Custom Bot

Move from user to creator. Custom bots have their own instructions, knowledge, and persona — persistent across every conversation, shareable with your team or students.

🤖

Bot Builder — Step by Step

▶

Custom bots are built in the Bot Garage — the toolbox icon in the top right navigation. Each bot gets its own instructions, knowledge base, and AI model.

Before You Open the Bot Builder

1

Define the Bot's Purpose

Write out: "You are an expert in [X]. Your role is to [Y]. Your audience is [Z]." This becomes your opening custom instruction.

2

Gather Knowledge Files

Collect documents into a folder first. Group related content into single text files — easier to update later. Convert PDFs to .txt where possible to save tokens.

3

Generate a Greeting

Draft what your bot says when someone opens it. Include 2–3 conversation starter examples so users know what to ask.

4

Generate an Icon Prompt

Ask an AI to write a Flux 2 Pro or Nano Banana prompt describing your bot's icon before you build. Have the image ready.

Inside the Bot Builder

  1. Toolbox icon (top right nav) → + Build Bot → Name and describe your bot
  2. Write Custom Instructions: role, audience, outcome goals, and step-by-step interaction process ("Step 1: First, ask the user… Step 2: After they respond, then…")
  3. Select your AI Model — bots have access to the full model library, including premium models not available in free chats
  4. Upload Knowledge — supported: .pdf, .docx, .txt, .png, .jpg (manage, delete, and update at any time)
  5. Test in the bot builder test window — reference uploaded materials, check response quality and tone
  6. Click Publish — copy the share link. Test it in a new tab before distributing.
Pro Tip — Iterative Improvement
After publishing, test your bot with real questions. If a response isn't ideal, create that question + ideal answer as a text file and add it to the bot's Knowledge Bank as an example. Use the "Add chat to knowledge bank" option on successful interactions.
Pro Tip — Feedback Loop
Add this to your custom instructions: "After every three interactions, ask the user to fill out a feedback form" and include a Google Form link. Continuous improvement without manual checking.
